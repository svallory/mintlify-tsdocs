The `Preview` component provides a dual-tab interface for showcasing React components alongside their source code. It's perfect for component documentation, allowing users to both see the component in action and copy the code to use it.

## Basic Usage

Import the component and wrap your example with both the code and the rendered output:

<Preview
  code={`<Preview
  code={\`<Button variant="primary">Click me</Button>\`}
>
  <Button variant="primary">Click me</Button>
</Preview>`}
>
  <div className="p-4 border border-gray-200 dark:border-gray-700 rounded">
    <button className="px-4 py-2 bg-primary text-white rounded hover:bg-primary-600">
      Click me
    </button>
  </div>
</Preview>

## Properties

<ResponseField name="code" type="string" required>
  The JSX code as a string. This is displayed in the Code tab and copied to clipboard when the copy button is clicked.
</ResponseField>

<ResponseField name="children" type="ReactNode" required>
  The rendered component to display in the Preview tab. This should be the actual component instance, not a string.
</ResponseField>

<ResponseField name="title" type="string">
  Optional title displayed above the preview content. Useful for labeling specific examples.
</ResponseField>

<ResponseField name="tooltipColor" type="string" default="#0D9373">
  Background color for the copy button tooltip. Defaults to Mintlify's primary green.
</ResponseField>

<ResponseField name="defaultTab" type="'preview' | 'code'" default="preview">
  Which tab should be active by default when the component loads.
</ResponseField>

<ResponseField name="onCopied" type="function">
  Callback function triggered when the copy button is clicked. Receives the copy result and the text that was copied.

  ```tsx
  (result: 'success' | 'error', textToCopy?: string) => void
  ```
</ResponseField>

<ResponseField name="className" type="string">
  Additional CSS classes to apply to the container element.
</ResponseField>

## Features

### Dual Tabs

The component provides two tabs:
- **Preview**: Shows the rendered component
- **Code**: Shows syntax-highlighted source code

### Copy to Clipboard

Both tabs share a copy button in the top-right corner that copies the source code. The button:
- Shows a tooltip on hover
- Provides visual feedback when clicked
- Works on both Preview and Code tabs
- Automatically hides if the browser doesn't support the Clipboard API

### Dark Mode

The component automatically adapts to Mintlify's theme:
- Light backgrounds in light mode
- Dark backgrounds in dark mode
- Proper contrast for all text and borders

## Code Formatting Tips

### Multiline Code

Use template literals with proper indentation for clean code display:

```jsx
<Preview
  code={`<ComplexComponent
  prop1="value1"
  prop2="value2"
>
  <ChildComponent />
</ComplexComponent>`}
>
  <ComplexComponent prop1="value1" prop2="value2">
    <ChildComponent />
  </ComplexComponent>
</Preview>
```

### Extracting Code

For complex examples, extract the code as a constant to keep things DRY:

```jsx
const exampleCode = `<TypeTree
  name="example"
  type="string"
  required={true}
/>`;

<Preview code={exampleCode}>
  <TypeTree name="example" type="string" required={true} />
</Preview>
```

### Escaping Backticks

If your code contains backticks, escape them properly:

```jsx
<Preview
  code={`<code>\\\`someCode\\\`</code>`}
>
  <code>`someCode`</code>
</Preview>
```

## Best Practices

<AccordionGroup>
  <Accordion title="Keep Code and Preview in Sync" icon="sync">
    The most common mistake is having the code string not match what you're rendering:

    ```jsx
    // ❌ Code and preview don't match
    <Preview code={`<Button>Click</Button>`}>
      <Button variant="primary">Click me</Button>
    </Preview>

    // ✅ Code and preview match
    <Preview code={`<Button variant="primary">Click me</Button>`}>
      <Button variant="primary">Click me</Button>
    </Preview>
    ```
  </Accordion>

  <Accordion title="Format Code for Readability" icon="code">
    Use proper indentation and line breaks for complex examples:

    ```jsx
    // ✅ Well-formatted
    <Preview
      code={`<TypeTree
  name="config"
  type="object"
  properties={[
    { name: "host", type: "string" },
    { name: "port", type: "number" }
  ]}
/>`}
    >
      {/* ... */}
    </Preview>
    ```
  </Accordion>

  <Accordion title="Add Titles for Context" icon="heading">
    Use titles to explain what the example demonstrates:

    ```jsx
    <Preview
      title="Required vs Optional Fields"
      code={/* ... */}
    >
      {/* ... */}
    </Preview>
    ```
  </Accordion>

  <Accordion title="Choose the Right Default Tab" icon="tab">
    For teaching, start with Preview. For reference, start with Code:

    ```jsx
    // Tutorial/guide - show the result first
    <Preview defaultTab="preview" code={/* ... */}>

    // API reference - show the code first
    <Preview defaultTab="code" code={/* ... */}>
    ```
  </Accordion>
</AccordionGroup>

## Browser Support

The copy-to-clipboard functionality requires:
- Modern browsers with Clipboard API support
- HTTPS connection (Clipboard API security requirement)

The component gracefully handles unsupported browsers by hiding the copy button.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Copy button not working" icon="clipboard">
    The Clipboard API requires HTTPS. If testing locally:

    1. Use `localhost` (works on HTTP)
    2. Or set up a local HTTPS certificate
    3. The component hides the button if unavailable
  </Accordion>

  <Accordion title="Code tab shows raw HTML" icon="code">
    Make sure you're passing JSX as a string, not rendered HTML:

    ```jsx
    // ❌ Wrong - renders HTML
    <Preview code={<div>Hello</div>}>

    // ✅ Correct - JSX as string
    <Preview code={`<div>Hello</div>`}>
    ```
  </Accordion>

  <Accordion title="Syntax highlighting not working" icon="paintbrush">
    The Preview component uses your site's configured syntax highlighter. Ensure:

    1. Syntax highlighting is enabled in Mintlify
    2. The `language-jsx` class is properly applied
    3. Your theme includes JSX/TSX support
  </Accordion>

  <Accordion title="Component not found" icon="triangle-exclamation">
    If you see `Cannot find module "/snippets/Preview.jsx"`:

    1. Ensure Preview.jsx exists in your `docs/snippets/` folder
    2. Run `mint-tsdocs` to auto-install the component
    3. Check that the import path is correct (should start with `/snippets/`)
  </Accordion>
</AccordionGroup>
