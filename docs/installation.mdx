## Package Installation

<CodeGroup>

```bash npm
npm install -D mintlify-tsdocs @microsoft/api-extractor
```

```bash bun
bun add -D mintlify-tsdocs @microsoft/api-extractor
```

```bash yarn
yarn add -D mintlify-tsdocs @microsoft/api-extractor
```

```bash pnpm
pnpm add -D mintlify-tsdocs @microsoft/api-extractor
```

</CodeGroup>

<Note>
  Both `mintlify-tsdocs` and `@microsoft/api-extractor` should be installed as dev dependencies since they're build-time tools.
</Note>

## System Requirements

<AccordionGroup>
  <Accordion title="Node.js Version" icon="node-js">
    - **Minimum**: Node.js 18.x

  <Accordion title="TypeScript Project" icon="code">

    - Compiled output (`.d.ts` files) available

    - `docs.json` configuration file
    - Mintlify CLI for local preview (optional)
  </Accordion>
</AccordionGroup>

## Project Setup

### 1. Package.json Scripts

Add convenient scripts to your `package.json`:

```json package.json
{
  "scripts": {
    "build": "tsc",
    "docs:extract": "api-extractor run --local",
    "docs:generate": "mintlify-tsdocs markdown -i ./temp -o ./docs/api --docs-json ./docs/docs.json --tab-name 'API' --group 'Reference'",
    "docs": "npm run build && npm run docs:extract && npm run docs:generate",
    "docs:dev": "cd docs && mintlify dev"
  }
}
```

<Tip>
  The `docs` script runs all three steps in sequence - build, extract, and generate.
</Tip>

### 2. TypeScript Configuration

Ensure your `tsconfig.json` includes:

```json tsconfig.json
{
  "compilerOptions": {
    "declaration": true,        // Generate .d.ts files
    "declarationMap": true,     // For better tooling support
    "outDir": "./dist",         // Output directory
    // ... other options
  }
}
```

<Warning>
  The `declaration: true` option is **required** - API Extractor needs the `.d.ts` files.
</Warning>

### 3. API Extractor Configuration

Create `api-extractor.json`:

```json api-extractor.json
{
  "$schema": "https://developer.microsoft.com/json-schemas/api-extractor/v7/api-extractor.schema.json",

  "mainEntryPointFilePath": "<projectFolder>/dist/index.d.ts",

  "apiReport": {
    "enabled": false
  },

  "docModel": {
    "enabled": true,
    "apiJsonFilePath": "<projectFolder>/temp/<unscopedPackageName>.api.json"
  },

  "dtsRollup": {
    "enabled": false
  },

  "messages": {
    "extractorMessageReporting": {
      "default": {
        "logLevel": "warning"
      }
    }
  }
}
```

<Info>
  API Extractor automatically replaces `<projectFolder>` and `<unscopedPackageName>` tokens based on your project.
</Info>

### 4. Mintlify Configuration

Ensure your `docs/docs.json` is set up:

```json docs/docs.json
{
  "$schema": "https://mintlify.com/schema.json",
  "name": "Your Project",
  "tabs": [
    {
      "name": "API Reference",
      "url": "api"
    }
  ],
  "navigation": [
    {
      "group": "Getting Started",
      "pages": ["introduction", "quickstart"]
    }
  ]
}
```

<Note>
  The `mintlify-tsdocs` tool will automatically add your API pages to this navigation structure.
</Note>

## Gitignore Configuration

Add these entries to your `.gitignore`:

```bash .gitignore
# API Extractor temporary files
temp/
*.api.json

# Build output (if not publishing)
dist/
lib/

# Mintlify local cache (optional)
.mintlify/
```

<Warning>
  **Do commit** your generated `docs/api/` folder - it's part of your documentation!
</Warning>

## Monorepo Setup

For monorepos with multiple packages:

<Steps>
  <Step title="Configure Each Package">
    Add `api-extractor.json` to each package that needs documentation
  </Step>

  <Step title="Centralized Docs">
    Create a single `docs/` folder at the root with shared `docs.json`
  </Step>

  <Step title="Package-Specific Scripts">
    Add docs scripts to each package's `package.json`:
    ```json
    {
      "scripts": {
        "docs:extract": "api-extractor run --local",
        "docs:generate": "mintlify-tsdocs markdown -i ./temp -o ../../docs/api/my-package --docs-json ../../docs/docs.json --tab-name 'API' --group 'My Package'"
      }
    }
    ```
  </Step>

  <Step title="Root Scripts">
    Add root-level scripts to generate all docs:
    ```json
    {
      "scripts": {
        "docs": "npm run docs:extract --workspaces && npm run docs:generate --workspaces"
      }
    }
    ```
  </Step>
</Steps>

## Verification

Verify your installation is working:

```bash
# 1. Build TypeScript
npm run build

# 2. Check that .d.ts files were created
ls dist/*.d.ts

# 3. Run API Extractor
npm run docs:extract

# 4. Check that .api.json was created
ls temp/*.api.json

# 5. Generate documentation
npm run docs:generate

# 6. Check that MDX files were created
ls docs/api/
```

<Check>
  If all these commands succeed, you're ready to generate beautiful API documentation!
</Check>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No .d.ts files generated" icon="file-circle-xmark">
    **Cause**: TypeScript isn't configured to generate declaration files.

    **Solution**: Add `"declaration": true` to your `tsconfig.json` compilerOptions.
  </Accordion>

  <Accordion title="API Extractor can't find entry point" icon="magnifying-glass">
    **Cause**: The `mainEntryPointFilePath` in `api-extractor.json` is incorrect.

    **Solution**: Ensure the path points to your compiled `index.d.ts` file. Check that:
    - Your TypeScript build completed successfully
    - The path matches your `outDir` in `tsconfig.json`
    - You're using the correct tokens (`<projectFolder>`)
  </Accordion>

  <Accordion title="No MDX files generated" icon="file-slash">
    **Cause**: No `.api.json` files found in input directory.

    **Solution**: Verify that:
    - API Extractor ran successfully
    - The `temp/` directory contains `.api.json` files
    - The `-i` flag points to the correct directory
  </Accordion>

  <Accordion title="docs.json not updated" icon="book-bookmark">
    **Cause**: File path incorrect or file has syntax errors.

    **Solution**:
    - Verify the `--docs-json` path is correct
    - Validate your `docs.json` syntax with a JSON validator
    - Ensure the file is writable (check permissions)
  </Accordion>
</AccordionGroup>
